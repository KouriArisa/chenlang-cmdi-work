# 优化说明

## 已处理的代码坏味道

- 避免使用 `data`、`info`、`manager` 等泛化命名
- 将排行榜排序规则提取为明确的比较函数
- 将玩家昵称、分页参数、成绩参数校验集中到统一模块
- 将魔法值提取为常量，例如分数、速度、棋盘尺寸、分页上限
- 将大逻辑拆成小函数，降低嵌套层级与分支复杂度

## AI 味代码清理

- 未堆砌解释性注释，只在必要处通过命名表达意图
- 未加入无实际价值的“万能工具类”或过度抽象
- 未添加静默 fallback、mock 成功路径或吞错逻辑

## 当前优化结果

- 前后端模块边界清晰
- 测试关注点明确
- 配置项可通过环境变量调整
- 本地与容器运行场景通过 `HOST` 明确区分
