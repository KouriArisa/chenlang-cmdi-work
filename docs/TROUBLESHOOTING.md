# 排障文档

## 1. `npm run build` 失败

### 现象

构建脚本提示源目录不存在或目标目录未创建。

### 原因

项目目录结构不完整，或手动删除了 `src/` 子目录。

### 解决方案

- 确认 `src/client` 与 `src/server` 目录存在
- 重新执行 `npm run build`

## 2. 启动后排行榜无法保存

### 现象

提交成绩时报文件写入错误。

### 原因

`DATA_DIR` 或 `DATA_FILE` 指向了不可写位置。

### 解决方案

- 确认运行用户对数据目录有写权限
- 优先使用默认目录 `./data`
- 使用 `./scripts/start.sh` 统一启动，脚本会自动准备目录和数据文件

## 3. Docker Compose 页面无法访问

### 现象

`http://localhost:8080` 返回 502 或连接失败。

### 原因

`app` 服务未通过健康检查，`proxy` 无法代理请求。

### 解决方案

- 运行 `docker compose ps` 查看 `app` 状态
- 运行 `docker compose logs app` 查看启动日志
- 确认 `PORT` 没有被错误改写

## 4. `npm test` 在 Node 25 下触发内部断言

### 现象

直接使用 `node:test` 跑 HTTP 集成测试时，Node 25 可能抛出内部断言错误。

### 原因

这是当前运行环境里的测试运行器兼容性问题，不是业务接口本身失败。

### 解决方案

- 保留 `node:test` 承载单元测试
- 将集成测试改为显式脚本执行
- 统一通过 `node ./scripts/test.mjs` 串联执行
